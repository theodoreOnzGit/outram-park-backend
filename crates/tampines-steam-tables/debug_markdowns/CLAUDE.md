# CLAUDE.md — Project Context for HRM Critical Flow Solver

## Project Overview

Rust-based 1D two-phase critical flow solver, transitioning from HEM
(Homogeneous Equilibrium Model) to HRM (Homogeneous Relaxation Model)
for nuclear safety applications (LOCA blowdown analysis).

## Test Data Generation: Backward Map

The test data is generated by working backwards from known throat conditions:

  Given: (p_throat, x_t)
    -> h_t = h_f(p_t) + x_t * h_fg(p_t)
    -> s_t = s_ph_eqm(p_t, h_t)
    -> G_crit = mass_flux_ps_eqm_throat(p_t, s_t)
    -> u_t = G_crit * v_t
    -> h_0 = h_t + 0.5 * u_t^2
    -> p_0 = p_hs_eqm(h_0, s_0)
    -> classify region of (p_0, h_0)

This generates self-consistent HEM test data. If the forward solver is
correct, it should recover the same (p_t, G) that was used to generate
the test case.

Boundary cases:
- x_t = 0.0: stagnation maps to Region 1 (subcooled) — tested by
  subcooled solver, not in-dome solver
- x_t = 1.0: stagnation maps to Region 2 (superheated vapour) — not
  currently tested by either solver

## Key Equations (Downar-Zapolski 1996)

### Conservation equations [1a]–[1d]:
  Mass:     d(rho)/dt + w*d(rho)/dz + rho*dw/dz = -rho*w/A * dA/dz
  Quality:  dx/dt + w*dx/dz = (x_bar - x) / theta
  Momentum: rho*dw/dt + rho*w*dw/dz + dP/dz = -tau - rho*g*cos(phi)
  Energy:   rho*dh/dt + rho*w*dh/dz - dP/dt - w*dP/dz = Q + w*tau

### Equation of state [2a]–[2b]:
  v = x * v_SG(P) + (1-x) * v_ML(P, h_ML)
  h = x * h_SG(P) + (1-x) * h_ML

### Relaxation equation [3]:
  Dx/Dt = (x_bar - x) / theta

### Equilibrium quality [5]:
  x_bar = (h - h_SL(P)) / (h_SG(P) - h_SL(P))

### Theta correlation [9] and [11]:
  theta = theta_0 * alpha^a * psi^b

  Low void (alpha < 0.14):
    theta_0 = 6.51e-4 s,  a = -0.257,  b = -2.24
    psi = (P_S(T_in) - P) / P_S(T_in)

  High void (alpha >= 0.14):
    theta_0 = 3.84e-7 s,  a = -0.54,   b = -1.76
    psi = (P_S(T_in) - P) / (P_c - P(T_in))

## Why HEM Fails at Bubble Point (Summary)

1. HEM forces x = x_bar instantaneously at the bubble point
2. This causes a discontinuous drop in speed of sound at alpha -> 0
3. The solver immediately enters supercritical flow regime at saturation
4. Critical mass flux is severely underpredicted for subcooled/near-
   saturated inlet conditions (up to 53% error per Downar-Zapolski Table 1)
5. Effect worsens for short L/D and low subcooling — exactly the
   conditions relevant to LOCA

Saha (1978) documents this across all available experimental datasets:
- HEM poor except at x_eq ~ 1 (high quality)
- Equilibrium slip models (Moody, Fauske) fail for L/D < 12
- Mechanistic models require arbitrary constants (bubble density varies
  by 5 orders of magnitude between models)
- Relaxation models identified as most promising path forward

## HRM vs HEM: What Changes in the Solver

| Component | HEM | HRM |
|---|---|---|
| Quality x | Algebraic lookup: x = x_bar(P,h) | Transported variable: solve dx/dt + w*dx/dz = (x_bar - x)/theta |
| Liquid state | Always saturated (SL) | Can be metastable (ML) |
| EOS | v = x_bar*v_SG + (1-x_bar)*v_SL | v = x*v_SG + (1-x)*v_ML |
| Speed of sound | Discontinuous at bubble point | Smooth transition |
| New closure needed | None | theta(alpha, psi) correlation |
| Source term treatment | N/A | Semi-implicit: explicit in x_bar/theta, implicit in x/theta |

## Numerical Implementation Notes

### Semi-implicit source term (recommended for HRM):
  x^(n+1) * (1/dt + 1/theta^n) = x^n/dt + x_bar^n/theta^n + advection

  This is unconditionally stable and automatically recovers:
  - HEM when theta -> 0 (x^(n+1) -> x_bar)
  - Frozen when theta -> infinity (x^(n+1) -> x^n)

### PIMPLE coupling sequence:
  1. Solve momentum -> w
  2. Solve energy -> h
  3. Solve pressure (PISO inner loops) -> P
  4. Update thermodynamics: h_SL(P), h_SG(P), x_bar, theta
  5. Solve relaxation equation -> x (semi-implicit source)
  6. Update mixture density from EOS using actual x
  7. Check convergence -> repeat or advance time step

### Steady-state form (for verification):
  w * dx/dz = (x_bar - x) / theta
  Characteristic relaxation length: L_relax = w * theta

## Model Hierarchy

| Model | Velocity | Thermal Equilibrium | theta | Complexity |
|---|---|---|---|---|
| HEM | Same | Instantaneous | 0 | Simplest |
| Frozen (HFM) | Same | Never | infinity | Simple |
| HRM | Same | Finite relaxation | 0.001–1 s | Moderate |
| Two-Fluid | Different | Separate energy eqns | N/A | Most complex |

Numerical equivalence (Downar-Zapolski p. 479):
  theta = 0.001 s  -->  recovers HEM
  theta = 1 s      -->  recovers Frozen (HFM)

## Caveats and Limitations

1. Theta correlation fitted to water only (Moby Dick data)
2. Sensitive to void fraction measurement (up to 10% variation
   from choice of data approximation method)
3. Two-regime split at alpha = 0.14 may cause discontinuity in theta
   — needs careful numerical handling (blending or clamping)
4. Extrapolation to superheated liquid region (before flashing inception)
   is acknowledged as uncertain by the authors
5. Correlation scatter is approximately +/-50% (Figure 5, p. 479)
6. Validated only for specific geometries (Moby Dick nozzle, Bolle
   safety valve nozzle) — Marviken validation is needed for large scale
